MÓDULO 4. ANÁLISIS Y DISEÑO DE SISTEMAS

1 - Escribe una breve definición de cada una de las palabras o conceptos relacionados.

Análisis de sistema: es una etapa de construcción de un sistema informático que implica relevar la información actual y proponer los rasgos generales de la solución futura.

Concepto de análisis: se refiere al examen minucioso y detallado de un asunto para comprender su naturaleza, características, estado y los factores involucrados.

Sistema: es un conjunto de elementos interrelacionados o componentes entrelazados entre sí para lograr un objetivo común.

Proyecto: es la planificación y ejecución de una serie de acciones para lograr un objetivo específico. Implica definir actividades, recursos y pasos a seguir.

Planificación: es la creación, asignación y distribución temporal de tareas para lograr un objetivo.


2 - ¿Cómo se realiza el análisis de sistema en el desarrollo de software, o cómo se aplica la planificación en la gestión de proyectos?

En el desarrollo de software, el análisis de sistemas implica identificar y documentar las necesidades del usuario y los requisitos del sistema a través de técnicas como entrevistas, cuestionarios y modelado, con el objetivo de definir claramente cómo debe funcionar el software. Esto se traduce en la creación de especificaciones detalladas que guían el diseño y desarrollo del sistema. Por otro lado, la planificación en la gestión de proyectos consiste en establecer un cronograma detallado, asignar recursos, definir el presupuesto y gestionar riesgos, para asegurar que el proyecto se ejecute de manera organizada, dentro del plazo y el presupuesto establecidos, mientras se comunica eficazmente con todos los stakeholders para cumplir con los objetivos del proyecto.


3 - Explora las relaciones entre los diferentes conceptos. Cómo el análisis de sistema se relaciona con el proyecto de sistema y la planificación.

El análisis de sistemas y la planificación están interrelacionados y se complementan en el desarrollo de software para garantizar el éxito del proyecto. El análisis de sistemas se enfoca en comprender y definir los requisitos y funcionalidades del software a través de la recolección y especificación detallada de las necesidades del usuario, creando una base sólida sobre la cual se construye el proyecto. Esta fase produce una documentación crítica que sirve como referencia para la planificación. Por su parte, la planificación toma los requisitos definidos durante el análisis y los traduce en un cronograma, presupuesto y asignación de recursos específicos, estableciendo cómo se llevará a cabo el desarrollo del sistema. Así, el análisis proporciona la dirección y el marco necesarios, mientras que la planificación organiza y coordina los esfuerzos para cumplir con los requisitos establecidos, asegurando que el proyecto se ejecute de manera eficiente y efectiva.


4 - Discute las ventajas y desventajas de cada concepto. Por ejemplo, ¿cuáles son las ventajas del análisis de sistema en comparación con otros enfoques?, o ¿cuáles son las desventajas de una mala planificación en un proyecto?

El análisis de sistemas ofrece la ventaja de proporcionar una comprensión profunda y detallada de los requisitos del software, lo que facilita la creación de soluciones que realmente satisfacen las necesidades del usuario y reduce el riesgo de malentendidos o fallos en el producto final. Sin embargo, puede ser costoso y llevar mucho tiempo, y si no se realiza correctamente, puede resultar en especificaciones incompletas o incorrectas. Por otro lado, una planificación efectiva asegura que el proyecto se mantenga dentro del presupuesto y el cronograma, prevé problemas potenciales y coordina los recursos de manera eficiente, aumentando las posibilidades de éxito. En contraste, una mala planificación puede resultar en retrasos, sobrecostos y una ejecución desorganizada, lo que compromete la calidad del producto final y la satisfacción del cliente.


5 - Crea un escenario ficticio donde debas aplicar el análisis de sistema, la planificación y otros conceptos relacionados. Desarrolla un caso detallado y propón soluciones basadas en los elementos del análisis y la fase de estudio de la planificación.

Imagina que una startup de e-commerce, EcoTrend, decide desarrollar una nueva plataforma de comercio electrónico para mejorar la experiencia de compra de sus clientes y gestionar mejor su inventario. En el análisis de sistemas, se realizan entrevistas con usuarios actuales y potenciales para identificar las funcionalidades necesarias, como la búsqueda avanzada de productos, un sistema de recomendaciones personalizadas y una interfaz intuitiva. El análisis también incluye la creación de diagramas de casos de uso y un prototipo de la interfaz para validar los requisitos con los stakeholders. En la fase de planificación, se define el alcance del proyecto, estableciendo entregables como el diseño de la plataforma, desarrollo del backend y pruebas de usabilidad. Se elabora un cronograma detallado con fases específicas y fechas límite, asignando recursos como desarrolladores y diseñadores, y se estima un presupuesto de $200,000 para cubrir todos los costos. Se identifican riesgos como posibles retrasos en la integración de sistemas y problemas de rendimiento, desarrollando un plan de mitigación con pruebas exhaustivas y soporte técnico adicional. Finalmente, se establece un plan de comunicación para mantener a todos los involucrados informados a través de reuniones semanales y actualizaciones regulares. Este enfoque garantiza que la plataforma cumpla con las expectativas del cliente, se desarrolle dentro del presupuesto y se lance a tiempo, maximizando el éxito del proyecto.
